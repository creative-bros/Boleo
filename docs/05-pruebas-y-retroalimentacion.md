# Manual 05: Pruebas y retroalimentación

## Objetivo

Facilitar que un tercero pruebe el sistema, detecte errores y envíe comentarios útiles para mejorar el portal.

## Qué se le debe enviar al evaluador

- URL del portal.
- Usuario y contraseña temporal.
- Rol asignado.
- Manuales de este directorio.
- Fecha límite para comentarios.

## Alcance sugerido de la prueba

Pedir que prueben al menos:

1. Inicio de sesión.
2. Navegación por módulos.
3. Alta o edición de datos.
4. Registro de pago.
5. Descarga de PDF.
6. Reserva de amenidad.
7. Alta de proveedor o tarea de mantenimiento.

## Formato sugerido para comentarios

Pedir que reporten cada observación así:

### 1. Módulo

Ejemplo: `Cobranza`, `Unidades`, `Amenidades`.

### 2. Acción realizada

Ejemplo: `Intenté registrar un pago`.

### 3. Resultado esperado

Ejemplo: `Esperaba ver el pago reflejado en el historial`.

### 4. Resultado real

Ejemplo: `El sistema guardó el pago, pero no cambió el saldo`.

### 5. Severidad

- Alta
- Media
- Baja

### 6. Evidencia

- Captura de pantalla
- Video
- Texto con pasos

## Checklist de validación

- El diseño carga correctamente.
- El usuario puede entrar sin error.
- Los permisos corresponden al rol.
- Se pueden registrar datos.
- Los datos quedan guardados.
- Los botones principales funcionan.
- Los PDFs descargan correctamente.
- Los textos se entienden con claridad.
- No hay errores 500 ni pantallas en blanco.

## Preguntas sugeridas para el evaluador

- ¿Qué parte fue más clara?
- ¿Qué parte fue más confusa?
- ¿Qué campo o proceso faltó explicar mejor?
- ¿Qué acción esperabas encontrar y no viste?
- ¿Qué mejorarías en diseño, texto o flujo?

## Entregable esperado del evaluador

Pedir que entregue:

- Lista de errores detectados.
- Lista de mejoras sugeridas.
- Capturas o videos.
- Comentario general de facilidad de uso.

## Recomendación interna

Conservar toda la retroalimentación en un solo documento o tabla para priorizar cambios por módulo.
